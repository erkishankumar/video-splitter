How to use & deploy to GitHub Pages
Local use: Just open index.html in Chrome or Firefox.

GitHub Pages:
1. Push both index.html and coi-serviceworker.js to your repo
2. Enable GitHub Pages in repo Settings → Pages
3. That's it — the service worker handles the SharedArrayBuffer headers automatically

Netlify / Vercel: Works out of the box. You can also add a _headers file (Netlify) or vercel.json with COOP/COEP headers instead of the service worker.

iOS limit: Files over ~250 MB may crash Safari on iPhone/iPad. This is an Apple WebAssembly memory restriction, not a bug in this tool.

Quality: Uses -c copy — no re-encoding, no quality loss. Splits happen at the nearest keyframe.
